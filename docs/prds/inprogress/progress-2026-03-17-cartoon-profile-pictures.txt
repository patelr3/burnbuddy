## Codebase Patterns
- Use `yarn test` (not `npx vitest`) to run API tests — local vitest binary is in node_modules/.bin
- Build shared package (`cd packages/shared && yarn build`) before API typecheck — API resolves @burnbuddy/shared from compiled dist/
- Fake timer tests: attach `expect().rejects` BEFORE `vi.advanceTimersByTimeAsync()` to avoid unhandled rejections
- Mock `fetch` with `vi.stubGlobal('fetch', fetchMock)` and clean up with `vi.unstubAllGlobals()` in afterEach
- Services use native `fetch` for HTTP calls (no axios dependency in the API package)
- Use real classes (not `vi.fn().mockImplementation`) in `vi.mock` factories for constructors — `resetAllMocks()` clears vi.fn implementations but classes survive
- `users-profile-picture.test.ts` and `users-delete-profile-picture.test.ts` are separate from `users.test.ts` — all import users router and need matching mocks for new dependencies
