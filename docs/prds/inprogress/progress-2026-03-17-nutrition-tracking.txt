## Codebase Patterns
- Shared package types go in individual files under `packages/shared/src/` and are re-exported from `index.ts`
- After changing shared package, run `cd packages/shared && yarn build` before API/web typecheck
- API tests: 503 tests across 24 files, all passing after US-003
- Firestore mock pattern: hoisted mockDocSnap (mutable `.exists`), mockDocRef (`.get()`/`.set()`), mockGetDb chain
- `food-search-cache.ts` wraps USDA calls with Firestore caching — use `cachedSearchFoods`/`cachedGetFoodDetails` in routes
- Node 24 has native `fetch` — use vi.stubGlobal('fetch', mockFn) to mock it in tests
- USDA nutrient mapping is in `services/api/src/services/usda-food-search.ts` — USDA_NUTRIENT_MAP constant
- Need `yarn install` in worktree before typecheck works (node_modules may not be present)
