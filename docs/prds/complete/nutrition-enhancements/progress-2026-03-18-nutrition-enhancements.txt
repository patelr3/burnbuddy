## Codebase Patterns
- Shared package must be built (`cd packages/shared && yarn build`) before API/web typecheck passes — they resolve from dist/
- Web pages at `apps/web/src/app/(main)/nutrition/recipes/` use `@burnbuddy/shared` imports for types and constants
- API route validation pattern: check input → return 400 with descriptive error → proceed with business logic
- API test pattern: vi.hoisted mocks + supertest + buildApp() — no emulators
- Auth cookie `auth_session` bypasses Next.js middleware for dev-browser testing
- Ingredient form fields (quantity/unit) only appear after selecting a food from USDA search
- yarn install needed before typecheck in fresh worktrees
- When adding a new Firestore collection, update ALL test files that mock getDb() (nutrition.test.ts AND nutrition-points.test.ts)
- Summary endpoint and evaluateNutritionPoints both query mealEntries + supplementEntries — keep both in sync
