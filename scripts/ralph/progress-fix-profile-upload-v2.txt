## Codebase Patterns
- `apiUploadFile()` in `apps/web/src/lib/api.ts` handles errors in priority order: specific status codes (413, 400) first, then generic `!res.ok` catch-all
- Server error responses include a JSON body with `error` field — always attempt `res.json()` before falling back to generic messages
- Use `.catch(() => null)` when parsing response JSON to safely handle non-JSON bodies
- sharp's prebuilt npm binaries bundle libvips with only libaom (AVIF). HEIC support requires building libvips from source with libheif + libde265. The bundled libheif is static (no plugin support).
- The Dockerfile uses `node:24-trixie-slim` (Debian 13) for access to modern libheif/libde265 packages
- When building libvips from source, enable `-Dintrospection=disabled` to avoid pulling in GObject introspection dependencies
- Custom-built vips uses dynamic modules (e.g., `vips-modules-8.17/vips-heif.so`). These must be copied to the runtime stage alongside `libvips*.so*`.

  - Custom-built libvips with `enable modules: true` creates dynamic modules like `vips-modules-8.17/vips-heif.so`. These MUST be copied to the runtime stage or HEIF format shows `input: false`.
  - Split large apt-get installs in Docker to avoid network timeouts in constrained environments.
  - Docker COPY resolves symlinks, so use `ldconfig` in runtime stage to recreate proper soname links.
