## Codebase Patterns
- UserProfile type is in `packages/shared/src/types.ts` — all workspaces reference this shared interface
- `yarn build` builds shared → web → api in order; `yarn typecheck` checks all three
- API tests use vitest with `vi.mock`/`vi.hoisted` pattern, no emulators needed
- For pure utility/lib tests, a simple `buildFakeDb()` stub is sufficient — no need for the full vi.hoisted/vi.mock pattern
- Firestore batch writes: mock with `mockBatch.mockReturnValue({ set: mockBatchSet, delete: mockBatchDelete, commit: mockBatchCommit })` and add `batch: mockBatch` to the `getDb()` stub
- When adding new mock functions to `vi.hoisted`, also add them to the destructured return AND to `beforeEach` reset defaults
- When existing profile data is needed in tests (e.g., for field checks), mock `exists: true` WITH `data: () => profileObj` — not just `exists: true`
- For username uniqueness checks in tests, chain mockGet responses: 1st for profile exists, 2nd for username doc exists, 3rd for fresh data after update
- For parallel Firestore queries (e.g., `Promise.all`), stack `mockCollectionGet.mockResolvedValueOnce()` — first call consumes first value, second call consumes second
