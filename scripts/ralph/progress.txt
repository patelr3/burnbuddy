## Codebase Patterns
- Use yarn workspaces for the monorepo (yarn 1.22.x available)
- All packages use TypeScript strict mode
- Always add node_modules/, dist/, .next/, *.tsbuildinfo to .gitignore before first commit
- `@burnbuddy/shared` is the shared types package — reference it as `"@burnbuddy/shared": "*"` in workspace deps
- services/api uses pino for structured JSON logging; log must include level, time, msg fields
- Run `git rm -r --cached <dir>` to unstage accidentally staged node_modules/dist before committing
