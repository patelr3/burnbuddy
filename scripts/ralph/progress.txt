## Codebase Patterns
- Always rebuild `packages/shared` (`yarn build` in shared dir) after changing types — the API resolves `@burnbuddy/shared` from compiled `dist/` output, not source
- Use `import { GROUP_WORKOUT_WINDOW_MS } from '@burnbuddy/shared'` for the group workout window constant
- Firestore queries: use simple single-field queries with in-memory filtering to avoid composite index requirements
- API tests: vi.hoisted + vi.mock + supertest + buildApp() pattern, run with `cd services/api && yarn test`
- Express route ordering: place named routes (e.g., `/partner-active`) BEFORE parameterized routes (e.g., `/:id`) to avoid conflicts
