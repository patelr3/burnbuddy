## Codebase Patterns
- Tailwind v4 CSS is properly bundled in Next.js standalone Docker output — the Dockerfile correctly copies `.next/static` to the runtime stage
- The deploy-web.yml workflow passes Firebase config as `--build-arg` to Docker build for NEXT_PUBLIC_* env vars
- FirebaseUI CSS is imported via `@import "firebaseui/dist/firebaseui.css"` in globals.css and gets bundled into the single Next.js CSS file
- Firebase Web client config (apiKey, authDomain, etc.) should be stored in Azure Key Vault as `firebase-web-*` secrets and fetched by the deploy workflow

- Firebase Admin SDK in the API container works with the minimal service account JSON (`{"type":"service_account","project_id":"buddyburn-beta"}`) — it successfully rejects invalid tokens
- Use custom domain URLs (buddyburn.arayosun.com) for browser testing — direct Azure FQDN URLs can timeout
- Both beta and production use the SAME Firebase project (`buddyburn-beta`) — confirmed via `firebase-service-account-json` in both KVs. Use `--env all` with setup script.
