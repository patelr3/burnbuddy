## Codebase Patterns
- jest-expo 55 requires jest ^29 (not 30) — check `node_modules/jest-expo/package.json` dependencies for version compatibility
- Use `setupFiles` (not `setupFilesAfterSetup`) in jest.config.js for setup scripts
- Mock `expo-modules-core` and `expo-notifications` in jest.setup.ts to prevent native module resolution errors
- `transformIgnorePatterns` must whitelist expo/react-native packages to allow babel transform in tests
- Run `yarn workspace mobile test` to execute mobile Jest tests
- Mock files in `src/__mocks__/` export individual mock fns + a `reset*()` helper; use `jest.mock('module', () => require('../__mocks__/file'))` pattern
- Firebase mock provides both `firebaseAuthModule` (for mocking 'firebase/auth') and `firebaseModule` (for mocking '../lib/firebase')
- Use `require()` inside `jest.mock()` factories to avoid hoisting scope errors — e.g., `jest.mock('firebase/auth', () => require('../../__mocks__/firebase').firebaseAuthModule)`
- For testing React context hooks, use `renderHook` from `@testing-library/react-native` with a wrapper component
- To test module-scope env var reads (e.g., `API_BASE`), use `jest.resetModules()` + re-apply `jest.mock()` + `require()` to reimport with new env
- Mock global `fetch` in `beforeEach` and restore in `afterAll` — create a `mockResponse()` helper to build Response-like objects
- For screen component tests that use `expo-web-browser` or `expo-auth-session`, mock them at the top of the test file (module-scope side effects)
- Mock `expo-auth-session/providers/google` useAuthRequest returning `[request, response, promptAsync]` tuple
- Spy on `Alert.alert` with `jest.spyOn(Alert, 'alert')` to verify error messages in components
