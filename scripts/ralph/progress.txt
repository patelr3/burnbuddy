## Codebase Patterns
- When mocking fire-and-forget service calls in vitest, re-apply `.mockResolvedValue()` in `beforeEach` after `vi.resetAllMocks()` — otherwise mock returns undefined and `.catch()` throws TypeError
- @types/express v5: req.params values are typed as `string | string[]` — use `req.params['name'] as string` to avoid TS2345 errors
- When modifying packages/shared/src/types.ts, run `yarn workspace @burnbuddy/shared build` before typechecking dependent workspaces (services/api imports dist/ declaration files)
- Firestore friendship pattern: use composite doc id `[uid1, uid2].sort().join('_')` for bidirectional relationships — enables O(1) lookup given both uids
- For Firestore query chains in vitest mocks, call `mockReturnThis()` on `where` and `limit` inside `beforeEach` (after `vi.resetAllMocks()`); use `mockResolvedValueOnce` on the terminal `get()` mock
- pino-opentelemetry-transport runs in a worker thread (like all pino transports), so OTel SDK initialized in main thread doesn't share state; use OTEL_LOGS_ENABLED=true to enable in environments with a collector
- OpenTelemetry SDK initialization must be the first import in the entry point (`import './instrumentation'` before all other imports) to ensure auto-instrumentation patches modules before they load
- apps/web logger: use `pino` with `browser: { asObject: true }` in Next.js — safe for both SSR (Node) and client-side (browser) without crashes
- `@opentelemetry/sdk-node` bundles sdk-trace-node, sdk-metrics, sdk-logs — only need to add it as a direct dep; `@opentelemetry/sdk-trace-node` still needed explicitly for `ConsoleSpanExporter` import
- Docker multi-stage builds for services/api: copy only packages/shared and services/api package.json into builder (skip apps/); yarn installs subset of workspaces; copy node_modules + packages/shared to runtime stage to preserve workspace symlinks
- Use yarn workspaces for the monorepo (yarn 1.22.x available)
- All packages use TypeScript strict mode
- Always add node_modules/, dist/, .next/, *.tsbuildinfo to .gitignore before first commit
- `@burnbuddy/shared` is the shared types package — reference it as `"@burnbuddy/shared": "*"` in workspace deps
- services/api uses pino for structured JSON logging; log must include level, time, msg fields
- Run `git rm -r --cached <dir>` to unstage accidentally staged node_modules/dist before committing
- Expo + yarn workspaces: convert root workspaces array to object with `nohoist` for react-native/expo packages
- apps/mobile/metro.config.js must set watchFolders and resolver.nodeModulesPaths for monorepo resolution
- Web support in Expo: install react-dom, react-native-web, @expo/metro-runtime via `npx expo install`
- Firebase CLI and Java are NOT available in the devcontainer — use vi.mock + vi.hoisted for firebase-admin unit tests instead of the emulator
- vitest is the test runner for services/api; use `vi.hoisted()` to create mock variables before `vi.mock()` factory executes (avoids temporal dead zone issues)
- Express Request type augmentation lives in src/types/express.d.ts (global namespace Extension); tsconfig "include": ["src"] covers it automatically
- firebase-admin emulator mode: set FIREBASE_AUTH_EMULATOR_HOST env var and initialize with just { projectId } — no credentials needed
- Firestore singleton: `getDb()` in services/api/src/lib/firestore.ts; mock via `vi.mock('../lib/firestore', () => ({ getDb: () => ({ collection: () => ({ doc: mockDocRef }) }) }))`
- vitest: use `vi.resetAllMocks()` (not `vi.clearAllMocks()`) in beforeEach to clear queued `mockResolvedValueOnce` values; then re-setup mock implementations in beforeEach
- New API routes go in services/api/src/routes/<resource>.ts and are registered in index.ts via `app.use('/<resource>', router)`
