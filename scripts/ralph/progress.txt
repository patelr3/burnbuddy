  - Notification data payload from push-notifications.ts: { type: 'WORKOUT_STARTED', uid: senderUid } — uid is sender, not a buddy/squad ID; look up the buddy by uid1/uid2 match in loadData results
  - Background/quit notification handling works automatically via system tray; no explicit code needed for this — expo-notifications handles it
