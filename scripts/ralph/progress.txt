## Codebase Patterns
- Bicep param files use `using './main.bicep'` syntax and follow naming pattern `main.<env>.bicepparam`
- API tests in `services/api` have pre-existing failures from stale `dist/` build artifacts (CommonJS import issue) — source `.ts` tests all pass
- Typecheck: `yarn typecheck` runs across all workspaces (mobile, web, shared, api)
- Deploy workflows use `azure/login@v2` with client-id/secret/tenant-id/subscription-id secrets and `az acr login` for container registry
- Docker images are tagged with `${GITHUB_SHA::8}` (first 8 chars of commit SHA)
- Container Apps are updated via `az containerapp update --name <app> --resource-group <rg> --image <image>`
- Beta and prod deploy as separate jobs with `needs:` to enforce sequential ordering (beta → prod)
- Next.js NEXT_PUBLIC_ vars must be passed as Docker build ARGs (baked at build time); the web Dockerfile has ARG directives for them
