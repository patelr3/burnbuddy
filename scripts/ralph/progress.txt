# Ralph Progress Log
Started: Thu Mar  5 07:32:41 AM UTC 2026

## Codebase Patterns
- Streak calculator now uses `GroupWorkout[]` as the sole input — no more per-member individual workout queries
- Streak reset uses a 7-day inactivity window — gaps of 1-6 days are tolerated, 7+ resets to 0
- Route test files use `vi.hoisted` + `vi.mock` pattern with collection-keyed mock routing in `getDb().collection(name)`
- When adding a new Firestore collection to route tests, add the mock query chain in `vi.hoisted`, add the collection routing in `vi.mock`, and add `mockReturnThis()` for `.where()` in `beforeEach`
- `calculateStreaks(groupWorkouts)` returns `{ burnStreak, supernovaStreak }` — both values are always equal since GroupWorkout implies all members participated
- Web app uses `apiGet`/`apiPut`/`apiPost`/`apiPatch` from `@/lib/api` for all authenticated API calls — avoid raw `fetch`
- Next.js `redirect()` from `next/navigation` provides server-side redirects for removed pages
