## Codebase Patterns
- After changing packages/shared/src/*, must run `cd packages/shared && yarn build` before API/web typecheck passes (they resolve from dist/)
- Root `yarn typecheck` fails due to pre-existing mobile errors — use workspace-specific typecheck
- Pre-existing test failure in users-profile.test.ts (streak assertion) — not related to our changes
- Use FieldValue.delete() from firebase-admin/firestore to unset Firestore fields when client sends null
- Health field validation is in validateHealthFields() in users.ts — reuse for new fields
- TypeScript interfaces can't be assigned to Record<string, unknown> — use destructuring (not delete loops) for type-safe field stripping
- stripHealthFields() in users.ts is the central place for health field privacy — add new health fields there too
- Test files are colocated with route files (services/api/src/routes/*.test.ts), not in __tests__ subdirectory
