# Ralph Progress Log — TLA+ Formal Specifications for BurnBuddy API
Started: 2026-03-07

## Codebase Patterns
- TLA+ specs extend Integers, FiniteSets, TLC directly — they do NOT extend Common.tla
- Use unordered sets {u1, u2} instead of ordered pairs for UID canonicalization (model values don't support <)
- Use decomposed state variables (separate sets/functions) instead of nested records to avoid TLC fingerprinting issues
- CHECK_DEADLOCK FALSE needed when specs have bounded time (AdvanceTime stops at MaxTime)
- Use SYMMETRY with Permutations() for state-space reduction on interchangeable model values
- cfg files use model values like {u1, u2} — these are auto-created by TLC
- Each domain spec is standalone with its own Init/Next/vars — composition requires a composite spec
- Cross-domain specs should use small bounds (2 users, 1 squad, 2 workouts) to keep state space manageable
