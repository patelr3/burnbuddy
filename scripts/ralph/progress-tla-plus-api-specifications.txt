# Ralph Progress Log — TLA+ Formal Specifications for BurnBuddy API
Started: 2026-03-07

## Codebase Patterns
- TLA+ specs live under `specs/tla/` with one subdirectory per domain
- `Common.tla` is the shared module — all domain specs should `EXTENDS Common`
- Each domain spec needs a `.tla` file and a `.cfg` TLC config file
- Typecheck is `yarn typecheck` — mobile workspace has pre-existing errors (Expo), ignore those; check api/web/shared workspaces individually if needed
- TLA+ files are not part of TypeScript typecheck — they are verified separately via TLC (`java -cp tla2tools.jar tlc2.TLC`)
- TLC run command: `cd specs/tla/<domain>/ && java -XX:+UseParallelGC -cp ../tla2tools.jar tlc2.TLC <Spec>.tla -config <Spec>.cfg -workers auto`
- TLA+ model values don't support `<` — use unordered sets `{u1, u2}` instead of ordered pairs for UID canonicalization
- For cross-domain dependencies (e.g., friendship before buddy), include simplified prerequisite actions rather than importing full specs
- Always parenthesize TLA+ arithmetic — `*` and `\div` have precedence conflicts
- Use `INVARIANT` in `.cfg` for state predicates (not `PROPERTY []Pred`)
- Model friendships as unordered sets `{u1, u2}` for natural symmetry; directional requests as records `[fromUid, toUid, status]`
- `tla2tools.jar` and `states/` directories are gitignored
- Entities with their own identity (squads, workouts) need a separate `CONSTANT` for IDs (e.g., `SquadId`) — keep the count small (1-2) to avoid state space explosion
- Use `CHOOSE s \in set : s.id = sid` for record lookup but always guard with an existence check first
- Defensive cleanup in spec actions (e.g., removing pending requests on delete) maintains state invariants even if the real API doesn't clean up — document as findings for VERIFICATION_REPORT
- Existing specs do NOT extend Common.tla — they extend standard modules (Integers, FiniteSets, TLC) directly and define types inline
- Common.tla has TLA+ parse errors (`--` not valid comment syntax) — never actually imported by TLC
- Use `{ pair \in SUBSET S : Cardinality(pair) = 2 }` for unordered pairs — avoids TLC parse errors with nested set comprehensions
- Time-based specs need `CHECK_DEADLOCK FALSE` in cfg — bounded time leads to terminal states
- Use 0 as sentinel for "not set" timestamp fields, start time at 1 so `endedAt > 0` invariant works
- Use addition-based range checks `~(a <= b + W /\ b <= a + W)` instead of subtraction to avoid Nat underflow
- TLC cannot compare records with non-records for fingerprinting — use decomposed state (separate set for existence + separate functions per field) instead of record-or-sentinel functions
- Use `CHOOSE u \in Set : TRUE` for irrelevant initial function values when domain elements may not yet be active
