# Ralph Progress Log — Site Performance Non-Dashboard Pages
Started: 2026-03-07

## Codebase Patterns
- Shared package build is broken on main (logger.ts needs `dom` lib for `console`). Compile types.ts individually: `npx tsc src/types.ts --declaration --outDir dist --target ES2022 --module CommonJS --esModuleInterop --strict`
- After changing packages/shared/src/types.ts, must rebuild dist/ before API typecheck passes. The API resolves @burnbuddy/shared from compiled dist/ output.
- Test mocks for Firestore use vi.hoisted + vi.mock pattern. Add `mockGetAll` to the hoisted mocks and `getAll: mockGetAll` to the getDb mock when testing batched reads.
- Firestore `db.getAll(...refs)` accepts spread of DocumentReferences and returns DocumentSnapshots in same order.
- `yarn install --frozen-lockfile` is needed in fresh worktrees before running tests.
- React Query migration pattern: import types (FriendsData, FriendWithProfile, etc.) from queries.ts, use hook return values instead of useState, use queryClient.invalidateQueries after mutations, use queryClient.setQueryData for optimistic updates.
- Skeleton components follow DashboardSkeleton pattern: `animate-pulse` wrapper div, `bg-gray-200 rounded` placeholder divs, `rounded-full` for avatar circles.
- Optimistic mutation pattern: `useMutation` with `onMutate` (cancelQueries + snapshot + setQueryData), `onError` (rollback from snapshot + setError), `onSettled` (invalidateQueries). Always `cancelQueries` before `setQueryData` to avoid race conditions.
