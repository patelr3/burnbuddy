## Codebase Patterns

- **Monorepo**: Yarn workspaces. Use `yarn workspace <name> <cmd>` or root scripts. Package names: `api`, `@burnbuddy/shared`, `web`.
- **Auth**: Clients use `firebase/auth` client SDK for tokens; API verifies via `firebase-admin` in `requireAuth` middleware (`services/api/src/middleware/auth.ts`). When `FIREBASE_AUTH_EMULATOR_HOST` is set, Admin SDK auto-routes to emulator.
- **User profiles**: Firebase Auth and Firestore are separate. Signup creates Auth account; `POST /users` creates Firestore profile. Users only appear in search after Firestore profile exists.
- **API structure**: Routes in `services/api/src/routes/`. `requireAuth` populates `req.user`. Firestore via `getDb()` from `services/api/src/lib/firestore.ts`.
- **Shared types**: `packages/shared/src/types.ts` — import as `import type { X } from '@burnbuddy/shared'`. Always build shared before typechecking: `yarn workspace @burnbuddy/shared build`.
- **API tests**: Vitest with `vi.mock` for Firebase — no emulator required. Run: `yarn workspace api test`.
- **Web app**: Next.js 14 App Router. All pages under `apps/web/src/app/`. API calls via `apiGet`/`apiPost` helpers in `apps/web/src/lib/api.ts`.
- **Infra**: Azure Bicep templates in `infra/`. Param files: `main.dev.bicepparam`, `main.beta.bicepparam`, `main.prod.bicepparam`. Resources named `buddyburn-<env>-*`.
- **CI/CD**: Two separate workflows — `deploy-api.yml` (paths: services/api/**, packages/shared/**, infra/**) and `deploy-web.yml` (paths: apps/web/**, packages/shared/**). Both deploy beta then prod.
- **Secrets in CI**: Azure credentials come from GitHub Actions secrets (AZURE_CLIENT_ID etc). App secrets (Firebase, test credentials) come from Azure Key Vault at runtime via `az keyvault secret show`.
- **Integration tests**: `tests/integration/api-smoke.sh` — set BASE_URL and AZURE_KEYVAULT_NAME env vars. Fetches test token from AKV then hits live API endpoints.
- **E2E tests**: Playwright under `tests/e2e/`. Set BASE_URL, INTEGRATION_TEST_EMAIL, INTEGRATION_TEST_PASSWORD env vars. `npx playwright test` from repo root.
