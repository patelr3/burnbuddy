## Codebase Patterns
- Firestore doc IDs for pair relationships use sorted composite keys: `${[uid1, uid2].sort().join('_')}` — see friends.ts and now burn-buddies.ts
- Tests use `vi.hoisted` + `vi.mock` for Firestore stubs; mock each collection's doc/query separately
- When adding a new Firestore read to a route, update ALL existing test cases that hit that route to mock the new read
- Build `packages/shared` before running `yarn typecheck` in `services/api`
- `requireProfile` middleware at `middleware/requireProfile.ts` — apply after `requireAuth` on social-action POST routes. Tests need `mockUsersDocGet.mockResolvedValueOnce({ exists: true, data: () => ({...}) })` for routes using it.
- Use `doc.create()` (not `doc.set()`) for atomic uniqueness guarantees — it throws code 6 (ALREADY_EXISTS) if the doc exists. See username.ts `tryReserveUsername()`.
- When removing batch writes from a route, update all test assertions from `mockBatchSet`/`mockBatchCommit` to `mockSet`
