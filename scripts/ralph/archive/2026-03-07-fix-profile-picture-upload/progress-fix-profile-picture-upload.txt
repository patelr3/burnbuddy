## Codebase Patterns
- Firebase Admin `initializeApp()` must include `storageBucket` for `admin.storage().bucket()` to work — without it, storage operations hang on an invalid bucket URL
- Use `getStorageBucket()` from `lib/storage.ts` for all Firebase Storage access — never call `admin.storage().bucket()` directly in routes
- When mocking storage in tests, mock `../lib/storage` module with `getStorageBucket` instead of `admin.storage().bucket`
- Pino logger is at `services/api/src/lib/logger.ts` — import it in route files for structured logging
- Anime filter runs 6 sequential sharp ops; posterize + Sobel X + Sobel Y can be parallelized with `Promise.all` for ~2x speedup
- API tests use `vi.hoisted` + `vi.mock` pattern — see `users-profile-picture.test.ts` for storage mock setup
- After changing packages/shared/src, must run `cd packages/shared && yarn build` before API typecheck
- Fake timers (`vi.useFakeTimers()`) conflict with supertest/multer I/O — use env-var-configurable timeouts with short real timeouts in tests instead
- React Native 0.83+ throws plain `Error` (not `DOMException`) on aborted fetch — check `err.name === 'AbortError'` without `instanceof DOMException`

## 2026-03-07T06:27 - US-001
- **What was implemented**: Fixed root cause of profile picture upload hang
- **Root cause**: `admin.storage().bucket()` was called without a bucket name, and `initializeApp()` had no `storageBucket` config, causing storage operations to use an invalid bucket URL and hang indefinitely
- **Files changed**:
  - `services/api/src/lib/firebase.ts` — Added `storageBucket` to all `initializeApp()` calls using `FIREBASE_STORAGE_BUCKET` env var with `{projectId}.appspot.com` default
  - `services/api/src/routes/users.ts` — Added try/catch error handling around anime filter, storage save, and signed URL generation; added pino logging for upload lifecycle events
  - `services/api/src/lib/anime-filter.ts` — Parallelized posterize + Sobel X + Sobel Y with `Promise.all` for ~2x speedup
- **Learnings for future iterations:**
  - Firebase Storage requires `storageBucket` in `initializeApp()` or explicit bucket name in `.bucket()` — without it, operations hang silently instead of throwing
  - The anime filter's posterize step and both Sobel convolutions are independent — safe to parallelize
  - Pre-existing typecheck failures exist in mobile (missing expo types) — API typecheck is the relevant check
  - All 309 API tests pass after changes
