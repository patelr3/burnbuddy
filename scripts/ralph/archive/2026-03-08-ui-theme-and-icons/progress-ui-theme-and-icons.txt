## Codebase Patterns
- Build shared package (`cd packages/shared && yarn build`) before running web typecheck
- Web typecheck: `cd apps/web && yarn typecheck`
- Use `yarn workspace web add <pkg>` to add web dependencies
- Tailwind v4 defines colors in `apps/web/src/app/globals.css` via `@theme` directive, NOT in tailwind.config.ts
- Firebase Auth widget styles are hardcoded in globals.css — update them when changing theme colors
- Success states use `bg-success`/`text-success` classes (green) — leave these when changing primary color
- Action buttons use `bg-primary` with `hover:bg-orange-600` — keep hover colors in sync with primary
- Dark theme buttons: use `bg-surface-elevated` (#2C2C2E), `border-[#3A3A3C]`, `text-white`, `hover:bg-[#3A3A3C]`
