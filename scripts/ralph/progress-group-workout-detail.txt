- API tests use `vi.hoisted()` + `vi.mock()` + `buildApp()` + `supertest` — always declare mocks before importing router
- Firestore doc mock pattern: `mockDocGet.mockResolvedValueOnce({ exists: true/false, data: () => obj })`

- Existing group-workouts.ts route uses `groupWorkouts` collection name in Firestore
- User profiles are in `users` collection, workouts in `workouts` collection
- `yarn test` in `services/api` runs vitest; `yarn typecheck` runs tsc --noEmit
- Must run `cd packages/shared && yarn build` before API typecheck (API resolves from dist/)
