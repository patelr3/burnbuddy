## Codebase Patterns
- Monorepo uses yarn workspaces with packages in `apps/*`, `packages/*`, `services/*`
- TypeScript strict mode is enabled in all packages (apps/web, services/api, packages/shared)
- API tests: only `src/` tests are reliable; `dist/` compiled tests fail with vitest CJS import error (pre-existing, not a blocker)
- Use `expo export --platform web` to verify Expo web bundle without needing a browser
- `@burnbuddy/shared` is the shared types/utilities package, imported by both web and api
- pino is used for structured JSON logging in services/api with OpenTelemetry transport
- `yarn build` runs `yarn workspaces run build` — every package needs a `build` script
- `yarn typecheck` runs `yarn workspaces run typecheck` — every package needs a `typecheck` script
- services/api uses Express.js with vitest for testing
- apps/web is Next.js 14 with app directory routing
- apps/mobile is Expo (React Native) — build script is `tsc --noEmit` (actual builds via EAS)
